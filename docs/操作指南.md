## 快速上手

部署完成后访问 `http://your-host:8081`，界面分为三个区域：

```
┌──────────┬─────────────────────────────────────┐
│  分组    │  [规则列表]                            │
│  树形    │  方法 │ 规则名/URL │ 状态 │ 粒度     │
│          ├─────────────────────────────────────┤
│          │  [编辑区域 / 规则编辑器]               │
└──────────┴─────────────────────────────────────┘
```

首次使用时自动创建一条默认 Mock 规则（标记为"请勿删除"），10 秒后自动持久化到磁盘。

### Dashboard 布局

- **左侧面板** — 分组树，筛选规则
- **右侧上半部** — Mock 规则列表，支持搜索
- **右侧下半部** — 规则编辑器，点击列表项开始编辑
- **底部按钮** — 保存 / 取消 / 测试 / 预览 JSON / 删除

## 分组管理

### 新建分组

1. 点击左侧分组栏的 **"新建"** 按钮
2. 填写信息：
   - **分组名称**：必填
   - **父级分组**：可选，支持嵌套
   - **URL 前缀**：可选，如 `/api/v1`。设置后该分组下的 Mock 编辑时自动填充前缀
   - **描述**：可选
3. 点击 **"确定"**

### 编辑分组

1. 鼠标悬停在分组上，点击 ✏️ 编辑图标
2. 修改信息后点击 **"确定"**

### 删除分组

1. 鼠标悬停在分组上，点击 🗑️ 删除图标
2. 确认删除（包含子分组时会有提示）

> **注意**：删除分组不会删除分组下的 Mock 规则，规则将变为"未分组"状态。

## Mock 规则

### 新建规则

1. 点击规则列表上方的 **"新建 Mock"** 按钮
2. 填写表单：

| 字段 | 说明 |
|------|------|
| 规则名称 | 便于识别的名称，可选 |
| 所属分组 | 选择分组后自动填充该分组的 URL 前缀 |
| 优先级 | 1-100，越小越优先，默认 5 |
| 请求方法 | GET / POST / PUT / DELETE / PATCH / HEAD / OPTIONS / ANY |
| URL 类型 | 精确匹配 / 路径匹配 / 路径正则 / 全 URL 正则 |
| URL 前缀 | 分组自动填充，也可手动修改 |
| URL | 必填，请求匹配的 URL |
| 匹配条件 | 可选，按请求头/查询参数/Cookie/请求体筛选 |
| 响应配置 | Direct / Fault / Proxy 三选一 |
| 响应延时 | 可选，单位毫秒 |

3. 点击 **"保存"**

### URL 前缀

分组设置了 URL 前缀后，Mock 编辑时会自动填充。效果：

- URL 输入框前显示蓝色前缀徽标 `/api/v1`
- 保存时自动判断是否需要拼接前缀
- 测试请求时自动拼接完整路径
- 列表中只显示相对路径

### 编辑规则

1. 点击规则列表中的任意一行即可编辑
2. 修改后点击 **"保存"**
3. 保存后延时 10 秒自动持久化到磁盘

### 响应模式

| 模式 | 说明 |
|------|------|
| **Direct** | 直接返回状态码 + 响应体 + 响应头 |
| **Fault** | 模拟网络故障（连接重置/空响应/畸形数据等） |
| **Proxy** | 代理转发到真实后端 |

#### Proxy 模式

勾选 Proxy 后填写：

- **代理目标**：真实后端地址，如 `http://real-backend:9090`
- **过滤前缀**：转发前去掉此前缀

例如：请求 `/api/v1/users` → 过滤前缀 `/api/v1` → 转发到 `http://backend:9090/users`

### 预览 JSON

1. 编辑规则时点击 **"预览 JSON"** 按钮
2. 弹出对话框分为两个标签页：
   - **美化预览**：带颜色的格式化 JSON，支持展开/折叠
   - **编辑**：VS Code 风格编辑器，可直接修改 JSON
3. 编辑后点击 **"保存"** 回填到表单并保存规则

### 删除规则

编辑时点击右下角 **"删除"** 按钮，确认后删除。

## 代理转发

### 全局代理

所有未被 Mock 匹配的请求转发到目标地址。

1. 切换到 **"代理转发"** 标签
2. 选择 **"全局代理"**
3. 填写 **"目标地址"**
4. 可选填写 **"去除前缀"**，转发时自动去掉此前缀
5. 点击 **"保存设置"**

### Catch-All Proxy

创建一个低优先级 Stub（priority=10）匹配所有请求并代理。

1. 切换到 **"代理转发"** 标签
2. 选择 **"低优先级 Stub 代理"**
3. 填写目标地址和前缀
4. 点击 **"保存设置"**

> 自定义 Mock 规则设置 priority 1-9 即可优先命中，未被命中的走兜底代理。

## 请求测试

### 发送测试请求

1. 编辑 Mock 规则时点击 **"测试"** 按钮
2. 弹窗中自动填充请求配置（URL 含前缀、Headers、Body）
3. 点击 **"发送"**

### 验证 Mock 命中

- 响应区域显示状态码、响应头、响应体
- 若命中 Mock，显示 **"Mock 命中"** 标签
- 可修改请求参数后重新发送

### 保存测试用例

点击 **"保存测试用例"** 将当前请求配置保存到 Mock 规则的 metadata 中，下次打开可直接加载。

### 代理开关

通过代理开关控制请求路径：
- **开启**：通过 Nginx 代理 → WireMock
- **关闭**：直连 `localhost:8080`（仅开发环境可用）

## 请求记录

查看 WireMock 接收到的所有请求：

- 实时记录最近 100 条请求
- 点击 **"开始轮询"** 每 5 秒自动刷新
- 修改查询条件后点击 **"刷新"** 重新获取
- 点击请求行查看详情（请求/响应 JSON 美化展示）
- 支持浅色/深色主题自适应

## JSON 编辑器

响应体编辑使用 VS Code 风格的 JSON 编辑器：

| 功能 | 操作 |
|------|------|
| 格式化 | 点击工具栏 **"格式化"** 按钮 |
| 缩进 | 按 `Tab` 插入 2 空格 |
| 语法错误 | 工具栏红色 ⚠ 提示，底部显示错误详情 |
| 行号 | 左侧自动生成行号 |
| 语法高亮 | 键名蓝色、字符串橙色、数字绿色、布尔/null 蓝色、括号紫色 |
